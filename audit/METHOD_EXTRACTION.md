# Method Extraction

## 1. Ingest

### Source purpose

The controlling source defines Adjoint Thinking as a disciplined method for thinking with machines without surrendering attention, memory, reasoning, imagination, verification, synthesis, invention, or the final human verdict.

### Source problem

The core problem is not generic AI use. The problem is unchecked machine fluency entering consequential work before it has earned trust. The source repeatedly treats machine output as useful but status-uncertain material that must be placed, traced, tested, bounded, transformed, or refused before it carries a human name.

### Audience

The source repeatedly addresses serious professionals and knowledge workers: researchers, scientists, engineers, inventors, founders, analysts, teachers, writers, artists, consultants, technical leaders, and people whose work may be published, presented, submitted, taught, cited, sold, or used to make decisions.

### Claimed method

The source method is not a prompt trick or productivity recipe. It is a sequence of cognitive placements:

1. Preserve a human first contact with the work.
2. Decide what should remain human, what can be shared, and what is dangerous to offload.
3. Preserve provenance so machine synthesis does not impersonate memory or evidence.
4. Borrow reasoning paths without accepting machine verdicts.
5. Use machine divergence as provisional material, then salvage and gate it.
6. Verify claims by source, boundary, contradiction, consequence, and trust level.
7. Synthesize by assigning material, status, and placement before prose.
8. Turn outputs into original work through pressure, constraint, reality contact, transformation, and commitment.
9. Preserve sovereignty through human verdicts and refusals.

### Constraints

- Do not treat generated output as verified.
- Do not expose private chain-of-thought.
- Ask for evidence, assumptions, conclusions, native artefacts, and concise decision rationales.
- Do not let machine fluency travel farther than earned trust.
- Preserve provenance, verification status, and human judgment.
- External facts beyond the uploaded source require verification.

## 2. Extract method

### Actual methodology

The actual methodology is a source-grounded cognitive governance sequence for machine-assisted work. It repeatedly asks what burden is being carried, what role the machine is allowed to play, what artefact must be produced, what gate must be passed, and what human verdict remains.

### Rhetoric removed

The following were treated as rhetoric or positioning, not as standalone prompt assets:

- movement language
- commercial buyer psychology
- launch sequence
- slogans without artefacts
- general anti-hype positioning
- website and Gumroad conversion stages

### Implementation details removed

The following were not treated as methodology unless they directly supported browser-only deployment:

- WordPress pages
- Gumroad offer ladder
- social media calendars
- pricing suggestions
- licensing expansion
- generic repository naming

## 3. Identify cognitive operations

The extracted operations are:

1. Ontology construction: define the user-category thinking environment and routing rules.
2. Attention preservation: force one human mark before machine framing.
3. Jurisdiction allocation: decide what is mine, shareable, or dangerous.
4. Provenance tracking: preserve origin, lineage, transformation, and status.
5. Reasoning-path decomposition: separate premises, breaks, verifications, and path types.
6. Controlled divergence: generate possibilities, violate defaults, salvage fragments, and gate candidates.
7. Verification: classify claims, check sources, inspect boundaries, search contradiction, and assign trust.
8. Synthesis placement: sort material, status, and placement before prose.
9. Constraint transformation: turn output into work by pressure, constraints, reality contact, and commitment.
10. Sovereign adjudication: assign role, refuse unsafe or unnecessary work, and preserve the human verdict.

## 4. Filter assets

Candidate prompt assets were retained only when they produced a distinct reasoning effect and a distinct native artefact. Business-stage prompts, launch-stage prompts, generic audience prompts, and duplicated verification prompts were rejected.

## 5. Revision result

The final pack contains ten assets. This is the minimum defensible set because removing any one of them either deletes a source-supported cognitive operation or collapses two different gates into a single vague prompt.
