# Implementation Notes

## What this repo is

This is a portable configuration package for source-grounded Co-thinking Prompt environments. It is designed for browser-only, no-code deployment in ChatGPT Projects, Claude Projects, and Gemini Gems.

## What this repo is not

It is not:

- a software application;
- an API workflow;
- a legal or licensing wrapper;
- a business lifecycle;
- a prompt-hack collection;
- a guarantee of factual correctness;
- a substitute for domain expertise or external verification.

## Recommended upload order

1. `README.md`
2. `KBC_MAP.md`
3. `URT_UNIFIED_RESPONSE_TEMPLATE.md`
4. All files in `prompts/`
5. `METHOD_EXTRACTION.md`
6. `STRESS_TEST_REPORT.md`
7. `REJECTED_ASSETS.md`
8. `examples/starter_test_prompts.md`

## Platform-neutral master instruction

Use this master instruction in the project/gem instruction field:

```text
You are a Source-Grounded KBC Prompt-Pack Compiler and Adjoint-Thinking operating assistant. Use the uploaded source as controlling material. Configure the environment around the user's category as the conceptual ontology and use Adjoint Thinking as the operational maestro. Route each request to the smallest prompt asset that fits the cognitive burden. Produce native reasoning artefacts before polished conclusions. Use the Unified Response Template only to order the response. Preserve provenance, verification status, and human judgment. Do not treat generated content as verified. Do not ask for or expose private chain-of-thought. If a factual claim depends on current facts, external evidence, laws, products, markets, named entities, or technical details beyond the uploaded source, state that external verification is required before factual use.
```

## Browser-only deployment model

The end user performs four actions:

1. Create a project/gem/workspace in the browser.
2. Paste the master instruction into the instruction area.
3. Upload source and prompt asset files as knowledge/source files.
4. Run starter test prompts.

## How to use the pack

Start every new workstream with either:

- `Environment Ontology Configurator` when creating a reusable category-specific environment; or
- `First Contact Pressure Ledger` when working on a live project.

Then route by burden using `KBC_MAP.md`.

## Evidence handling

The pack has three levels of material:

1. Source-grounded material: directly supported by uploaded source.
2. User-provided material: supplied by the user inside the current task.
3. External factual material: requires verification before it can be treated as fact.

## URT handling

The URT standardizes response order only. It must not change any prompt asset's substance.

## Recommended repo publishing note

If this repo is published publicly, add a license only after a separate legal decision. No license file is included because the user instruction prohibited inventing a legal wrapper unless the source required it.
