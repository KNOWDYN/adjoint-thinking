# Final KBC Map

The KBC map defines the minimum complete prompt-pack for source-grounded Adjoint-Thinking environments.

| # | Official name | Core reasoning operation | Purpose | Input slot | Native artefacts | Terminal outcome | Difference from other assets |
|---|---|---|---|---|---|---|---|
| 0 | Environment Ontology Configurator | Ontology construction and routing | Convert a user category and source corpus into a usable thinking environment | User category, work context, source files, output goals | User-Category Ontology, Environment Charter, Routing Table, Evidence Boundaries | A deployable project/gem/project instruction spine | It configures the environment before any specific work begins |
| 1 | First Contact Pressure Ledger | Attention preservation and pressure localization | Preserve the user's first encounter with the work before machine framing | Live project, difficulty, current pressure, attempted work | Human Mark, Pressure Statement, Attention State, Next Allowed Machine Role | Machine assistance enters after human contact | It protects the beginning of thought rather than classifying offload zones |
| 2 | Cognitive Jurisdiction Map | Jurisdiction allocation | Decide what must remain human, what can be shared, and what is dangerous | Project, desired machine help, intended use, risk level | Mine / Shareable / Dangerous Map, Permission Rules, Offload Boundary | Clear machine jurisdiction before execution | It governs permission; it does not inspect sources or generate ideas |
| 3 | Living Dossier Provenance Audit | Provenance tracking and memory repair | Prevent machine synthesis from impersonating memory or evidence | Notes, sources, summaries, drafts, claims, machine outputs | Living Dossier, Provenance Marks, Boundary List, Disagreement Chamber | A traceable project archive | It protects memory and lineage, not reasoning validity or publication readiness |
| 4 | Reasoning Path Ledger | Reasoning-path decomposition | Borrow useful reasoning paths without accepting machine verdicts | Claim, argument, explanation, decision, model output | Path Options, Premise Ledger, Break Test, External Verification List | Human-owned reasoning path with no borrowed verdict | It works on reasoning structure, not factual source support alone |
| 5 | Controlled Divergence Chamber | Divergence, analogy, violation, salvage, gating | Generate non-final possibilities beyond habit while keeping them provisional | Project pressure, constraints, forbidden cheapening, desired direction | Laboratory Notebook of Impossible Things, Violation List, Salvage List, Test Gates | Candidate possibilities ready for later verification or transformation | It produces provisional material, not verified claims or final choices |
| 6 | Verification Receipt | Validation, falsification, and trust classification | Check AI-assisted work before it carries a name or consequence | AI output, intended use, claims, sources, consequence level | Claim Extraction Sheet, Source Support Check, Boundary Audit, Contradiction Search, Trust Label, Receipt | Use, revise, deepen, or reject based on earned trust | It assigns trust status; it does not synthesize the whole work |
| 7 | Synthesis Wall | Integrative placement | Build structure from fragments without hiding seams | Claims, sources, contradictions, boundaries, machine syntheses, human judgments | Material Table, Status Table, Placement Table, Structural Pressure Statement | A defensible architecture before prose | It places checked and provisional material; it does not invent candidates |
| 8 | Invention Forge Transformation Log | Constraint transformation and reality contact | Turn output into original work through pressure, constraints, tests, and commitment | Candidate output, pressure, constraints, prototype/test options | Forge Stations, Constraint Conflict, Reality Contact Plan, Transformation Log | Candidate transformed into accountable work or abandoned | It tests and transforms candidates; it does not simply generate options |
| 9 | Sovereignty Receipt | Adjudication, refusal, role control | Preserve the final human verdict and refusal rights | Proposed output, decision, role of machine, cost, consequence | Role Assignment, Human Verdict, Refusal Ledger, Sovereignty Receipt | A decision the user can own, refuse, or defend | It governs final commitment and life/work boundary, not just correctness |

## Router

Use this router before selecting any full prompt.

| User burden | Use asset |
|---|---|
| I need to configure a reusable AI environment for a category of users | Environment Ontology Configurator |
| I am tempted to ask the machine before I have faced the work | First Contact Pressure Ledger |
| I do not know what AI should or should not touch | Cognitive Jurisdiction Map |
| My notes, sources, and outputs are untraceable | Living Dossier Provenance Audit |
| I need help analyzing or testing a claim without borrowing the verdict | Reasoning Path Ledger |
| I need more possibilities, analogies, names, mechanisms, or directions | Controlled Divergence Chamber |
| I need to check claims, citations, summaries, reasoning, or public-ready text | Verification Receipt |
| I have fragments but no defensible structure | Synthesis Wall |
| I have an output or idea that must become original work | Invention Forge Transformation Log |
| I need to decide, refuse, sign, publish, release, or preserve human control | Sovereignty Receipt |

## Hard pack rule

No prompt asset may treat output as publication-ready unless provenance, verification status, and human judgment are visible.


## Release classification note

For the release-grade package, Asset 0 is treated as a meta-configuration asset. Assets 1–9 are the source-native Adjoint Thinking co-thinking operations. This keeps deployment setup separate from the method operations while preserving the full source-grounded sequence.
