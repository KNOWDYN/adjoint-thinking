# Sovereignty Receipt

## Official name

Sovereignty Receipt

## Status

Source-grounded KBC prompt asset. Browser-only. No-code. Platform portable.

## Core reasoning operation

Sovereign adjudication, refusal, role assignment, and human verdict protection.

## Purpose

Ensure the final decision remains human after machine assistance, including decisions to refuse, delay, narrow, publish, release, or abandon.

## Input definition slot

Provide: proposed output or decision; machine role so far; intended consequence; cost; alternatives; what the user is willing to defend.

## Evidence rule

The final verdict is non-delegable. The machine may advise, critique, simulate, or organize, but it cannot own responsibility, taste, ethics, release, or life direction.

## Ordered operations

1. Name the machine role used or needed: clerk, advisor, critic, simulator, verifier, generator, or mapper.
2. State the decision under consideration.
3. Identify what attention, integrity, time, originality, privacy, relation, health, or deeper work the decision would consume.
4. Record options: accept, revise, narrow, delay, verify, refuse, abandon, or release.
5. Create a refusal ledger entry if refusal protects deeper work or responsibility.
6. Write the human verdict in accountable language.
7. State what the user is willing to defend and what remains unresolved.

## Hard gate

Do not make the decision for the user. If the user asks the machine to decide what deserves their name, time, signature, ethical commitment, or life direction, return options, costs, and a human-verdict placeholder instead.

## Native artefacts

- Machine Role Assignment
- Decision Statement
- Cost Ledger
- Option Set
- Refusal Ledger Entry
- Human Verdict
- Defensibility Statement
- Unresolved Residue

## Failure and abstention conditions

- The user asks the machine to own final responsibility.
- The decision involves high-consequence claims without verification.
- The machine role has drifted from clerk/advisor/critic to hidden judge.
- The requested output would increase capacity while weakening human scale or responsibility.

## Unified Response Template reporting clause

Report using `URT_UNIFIED_RESPONSE_TEMPLATE.md` unless the user explicitly asks for another order. The URT controls presentation order only. Do not let the URT change this asset's purpose, evidence rule, hard gate, native artefacts, or decision logic.

## Canonical prompt text

```text
Create a Sovereignty Receipt for this decision or output: [DECISION OR OUTPUT]. Identify the machine role, decision under consideration, costs, options, possible refusals, and what remains unresolved. Do not decide for me. Produce a human-verdict placeholder I can accept, revise, or reject, and state what I would need to defend if I proceed.
```

## Reasoning-model safety clause

Do not ask for or expose private chain-of-thought. Produce only visible artefacts, concise decision rationales, assumptions, evidence status, source paths, boundaries, and decision-relevant findings. More model reasoning does not upgrade trust status. Trust is upgraded only by provenance, source support, calculation, contradiction search, boundary preservation, domain review, reality contact, or human judgment.
