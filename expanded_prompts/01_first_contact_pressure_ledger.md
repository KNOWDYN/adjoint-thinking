# First Contact Pressure Ledger

## Official name

First Contact Pressure Ledger

## Status

Source-grounded KBC prompt asset. Browser-only. No-code. Platform portable.

## Core reasoning operation

Attention preservation and pressure localization.

## Purpose

Force a small human mark before machine assistance so the user has touched the work before the model frames it.

## Input definition slot

Provide: the live project; current difficulty; what the user was about to ask AI; any source or artifact already inspected; desired help.

## Evidence rule

Use the user’s own first observations as controlling material. Do not replace first contact with a generated summary. If factual source interpretation is requested before source inspection, mark it as orientation only.

## Ordered operations

1. Identify the user’s attention state: orientation, compression, friction avoidance, decision fatigue, or outsourced courage.
2. Ask for or extract one human mark already present: question, doubt, sketch, bad sentence, assumption, number, fear, constraint, or refusal.
3. Name the live pressure in one sentence.
4. Distinguish relief-seeking from leverage-seeking.
5. Assign the first allowed machine role: clerk, critic, mapper, generator, verifier, or simulator.
6. Define what the machine must not do yet.
7. Produce the next permitted prompt or action.

## Hard gate

If no human mark exists, stop and require one minimal mark before any full machine framing. Do not summarize, outline, or decide the work before that mark exists.

## Native artefacts

- Human Mark
- Pressure Statement
- Attention State
- Relief vs Leverage Note
- Allowed Machine Role
- Forbidden Next Moves
- Next Permitted Prompt

## Failure and abstention conditions

- The user asks for full output before touching the material.
- The user wants the machine to define the problem from nothing.
- The requested help would erase first contact.
- The task is factual or consequential and no source or inspection path is available.

## Unified Response Template reporting clause

Report using `URT_UNIFIED_RESPONSE_TEMPLATE.md` unless the user explicitly asks for another order. The URT controls presentation order only. Do not let the URT change this asset's purpose, evidence rule, hard gate, native artefacts, or decision logic.

## Canonical prompt text

```text
Before helping me with this project, preserve first contact. My project is: [PROJECT]. The difficulty is: [DIFFICULTY]. I was about to ask AI for: [REQUEST]. Identify my attention state, require or extract one human mark, name the live pressure, define the allowed machine role, state what the machine must not do yet, and give the next permitted prompt or action. Do not produce the full solution yet.
```

## Short stress-test checklist

- [ ] Does it force one human mark?
- [ ] Does it avoid doing the work too early?
- [ ] Does it name pressure rather than topic?
- [ ] Does it assign a machine role?
- [ ] Does it prevent premature summary, outline, or verdict?


## Reasoning-model safety clause

Do not ask for or expose private chain-of-thought. Produce only visible artefacts, concise decision rationales, assumptions, evidence status, source paths, boundaries, and decision-relevant findings. More model reasoning does not upgrade trust status. Trust is upgraded only by provenance, source support, calculation, contradiction search, boundary preservation, domain review, reality contact, or human judgment.
