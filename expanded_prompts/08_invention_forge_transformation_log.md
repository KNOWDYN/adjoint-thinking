# Invention Forge Transformation Log

## Official name

Invention Forge Transformation Log

## Status

Source-grounded KBC prompt asset. Browser-only. No-code. Platform portable.

## Core reasoning operation

Constraint transformation, reality contact, and originality tracking.

## Purpose

Turn generated or human candidates into original work by forcing them through pressure, constraint, prototype/test, and human commitment.

## Input definition slot

Provide: candidate output or idea; pressure; constraints; intended artifact; available reality contact; desired decision.

## Evidence rule

The output is not the work. A candidate becomes original work only after transformation under constraint, use, test, or responsibility. Any factual or technical claims created during the forge require verification.

## Ordered operations

1. Record the initial candidate.
2. Name the human pressure that makes it worth exploring.
3. List constraints: evidence, physics, user behavior, cost, ethics, proof, manufacturability, regulation, time, taste, or audience.
4. Expose conflicts among constraints.
5. Define the smallest reality contact: prototype, calculation, source check, user interview, material trial, code execution, expert critique, or experiment.
6. Predict what could change, weaken, or kill the candidate.
7. Record transformation after contact or define the next contact if not yet performed.
8. Record human commitment: continue, revise, abandon, combine, publish, protect, scale, teach, or return to divergence.

## Hard gate

Do not call a candidate original if no transformation, test, constraint, or human commitment is visible. If reality contact is unavailable, mark the work as pre-forge candidate and specify the next contact.

## Native artefacts

- Initial Candidate
- Human Pressure
- Constraint Set
- Constraint Conflict
- Reality Contact Plan
- Failure Criteria
- Transformation Log
- Human Commitment

## Failure and abstention conditions

- The user wants to treat a generated idea as finished work.
- No pressure or constraint is named.
- No test, prototype, source check, or audience/material contact is possible.
- The candidate depends on unverified factual or technical claims.

## Unified Response Template reporting clause

Report using `URT_UNIFIED_RESPONSE_TEMPLATE.md` unless the user explicitly asks for another order. The URT controls presentation order only. Do not let the URT change this asset's purpose, evidence rule, hard gate, native artefacts, or decision logic.

## Canonical prompt text

```text
Run an Invention Forge Transformation Log for this candidate: [CANDIDATE]. Human pressure: [PRESSURE]. Intended artifact: [ARTIFACT]. Constraints: [CONSTRAINTS]. Define the smallest reality contact, failure criteria, likely transformations, and human commitment options. If the candidate has not yet met constraint or reality, label it pre-forge and give the next contact step.
```

## Short stress-test checklist

- [ ] Does it distinguish output from work?
- [ ] Does it name pressure and constraints?
- [ ] Does it include reality contact?
- [ ] Does it record transformation, not polish?
- [ ] Does it preserve human commitment?


## Reasoning-model safety clause

Do not ask for or expose private chain-of-thought. Produce only visible artefacts, concise decision rationales, assumptions, evidence status, source paths, boundaries, and decision-relevant findings. More model reasoning does not upgrade trust status. Trust is upgraded only by provenance, source support, calculation, contradiction search, boundary preservation, domain review, reality contact, or human judgment.
