# Controlled Divergence Chamber

## Official name

Controlled Divergence Chamber

## Status

Source-grounded KBC prompt asset. Browser-only. No-code. Platform portable.

## Core reasoning operation

Controlled divergence, analogy, violation, salvage, and gating.

## Purpose

Generate possibilities beyond the user’s first habits while keeping all outputs provisional until later gates.

## Input definition slot

Provide: project pressure; current safe/default direction; desired output class; constraints; what must not be cheapened; later test gate if known.

## Evidence rule

Generated possibilities are private material. They are not truth, evidence, final taste, or publishable claims. Any factual or source-like output requires verification before use.

## Ordered operations

1. State the project pressure, not just the topic.
2. Name the current average/default path.
3. Generate variations that intentionally violate assumptions, genre habits, or expected paths.
4. Mark each useful violation and why it matters.
5. Extract salvage fragments from failed or impossible ideas.
6. Assign a future test gate: source check, prototype, sketch, calculation, audience test, craft revision, ethical review, or experiment.
7. Select candidates for the next work act without treating them as final.

## Hard gate

Do not present divergent outputs as true, original, verified, or ready. If no project pressure is provided, stop and request or infer a provisional pressure before generating options.

## Native artefacts

- Project Pressure Statement
- Default Pattern Map
- Violation List
- Salvage List
- Laboratory Notebook of Impossible Things
- Test Gates
- Candidate Shortlist

## Failure and abstention conditions

- The user asks for factual claims, citations, or professional advice in a divergence session.
- The output becomes abundance without selection.
- No pressure, constraint, or gate is available.
- The user asks the machine to choose taste or final direction.

## Unified Response Template reporting clause

Report using `URT_UNIFIED_RESPONSE_TEMPLATE.md` unless the user explicitly asks for another order. The URT controls presentation order only. Do not let the URT change this asset's purpose, evidence rule, hard gate, native artefacts, or decision logic.

## Canonical prompt text

```text
Open a Controlled Divergence Chamber for this project: [PROJECT]. The pressure is: [PRESSURE]. The current safe/default direction is: [DEFAULT]. Generate provisional variations that break habits without cheapening the work. For each useful direction, mark the violation, salvage fragment, and future test gate. Do not treat any output as true, verified, final, or ready.
```

## Short stress-test checklist

- [ ] Does it name pressure before generating?
- [ ] Does it produce violations rather than generic ideas?
- [ ] Does it salvage fragments from failures?
- [ ] Does every candidate have a gate?
- [ ] Does it avoid pretending divergence is final creativity?


## Reasoning-model safety clause

Do not ask for or expose private chain-of-thought. Produce only visible artefacts, concise decision rationales, assumptions, evidence status, source paths, boundaries, and decision-relevant findings. More model reasoning does not upgrade trust status. Trust is upgraded only by provenance, source support, calculation, contradiction search, boundary preservation, domain review, reality contact, or human judgment.
